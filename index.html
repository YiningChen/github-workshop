What is git?

  • version control system
  • not the same thing as github!
  • easy to use! (well, most of the time...)

Why is git awesome?

  • you can easily see and edit your codebase at each saved change (or "commit")
  • you can branch off from the codebase and make changes without risking screwing things up
  • you can merge branches together, so it's easier to collaborate with people! ‚ò∫Ô∏è

Setting up a repo

  • A repository contains all of the commits (or changes) to your codebase
  • Creating A Repo: follow the directions here!
  • Forking A Repo: click fork!
  • Clone (i.e. download) the repo!: 
    git clone git@github.com:username/repo.git

Getting code on to the repo:

// after you've made some changes, see which files you've changed:
git status

// mark all of your changes to be committed
git add .

// actually commit the changes,
// and give yourself a helpful reminder what the change is!
git commit -m 'I am a helpful message'

// right now this commit is still local (only on your computer)
// push this change online!
git push

Seeing the changes you've made:

  • see which files you've changed:
    git status
  • see the actual changes
    git diff

Adding changes to be committed:

  • add all changes: 
    git add .
  • add just a file or folder: 
    git add path/to/file
  • ah! nvm! I don't want any of these changes!üò± 
    git reset

Commiting and Pushing the change:

  • Commit with a message: 
    git commit -m 'helpful message'
  • Push to remote origin: 
    git push origin HEAD
  • Now someone else can 
    git pull
    to get your changes!

Oh shoot...I don't want those changes but already pushed:

  • get the commit hash from the log 
    git log
  • revert that commit 
    git revert 784fa95905f4f9f409bb567fe3a7ca48998825fd
  • (don't forget to push!)

git status is your best friend

  • always safe to use
  • see which files changed
  • see which files haven't been added
  • see which files are added but NOT committed

What is HEAD?

  • HEAD is a reference to the current commit
  • In most situations, it refers to the current branch
  • If the current branch is master, this:
    git push origin HEAD
  • ...is shorthand for this:
    git push origin master

Merging

  • switch to branch you want to merge to
    git checkout branch-to-merge-to
  • merge other branch, but don't commit so you can test the merge first!
    git merge --no-commit branch-i-want-to-merge-with
  • then commit
    git commit -m "okay we're good üëç "
  • pushhhhhh!
    git push origin HEAD

a few cool tricks

// checkout the previous branch I was on
git checkout -

// git is not letting me switch branches 
// because I have uncommitted changes
// let's stash them!
git stash

// wait I want those changes back!
git stash pop

// I want my team member's commit 
// because his/her change will be helpful
git cherry-pick <commit-hash>
